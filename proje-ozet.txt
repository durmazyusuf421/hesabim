5. BILINEN HATALAR / SORUNLAR
==============================================================

  - Yevmiye senkronizasyonu: cari_hareketler tablosunda sirket_id
    filtresi yok (tum sirketlerin hareketlerini cekiyor)
  - Bilanco dengesi: Kasa girisi karsi hesap olarak 391 (Hesaplanan KDV)
    kullaniliyor, bu muhasebesel olarak yanlis olabilir
  - Fatura toplu PDF: Cok sayida fatura secildiginde (50+) tarayici
    yavaslamasi yasanabilir
  - Portal raporlarinda masraf verisi: perakende_satislar tablosundan
    alis faturasi verisi cekilemiyor (sadece satis var)
  - Gelir tablosu: Satis iadesi ve iskonto su an 0 olarak sabit,
    henuz iade/iskonto modulu yok
  - Mizan/Bilanco: Yevmiye kayitlari senkronize edilmemisse
    bos gorunuyor, kullaniciya uyari gosteriliyor ama yonlendirme yok


==============================================================
  TEKNIK BILGILER
==============================================================

Framework:      Next.js 14+ (App Router)
UI:             Tailwind CSS + Ozel CSS degiskenleri
State:          React useState/useMemo (hook bazli, global state yok)
Auth:           Supabase Auth + localStorage senkronizasyonu
Database:       Supabase (PostgreSQL)
RLS:            get_my_sirket_id() fonksiyonu ile satir bazli guvenlik
Charts:         Recharts
PDF:            jsPDF
Excel:          xlsx (SheetJS)
Portal:         Cift portal: TOPTANCI (/) ve PERAKENDE (/portal)
Deployment:     Vercel (tahmini)
